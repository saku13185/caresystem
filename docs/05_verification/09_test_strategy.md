# 하네스 엔지니어링 5대 검증 테스트 전략서 (Harness Engineering Test Strategy)

본 문서는 예방적 돌봄 AI 에이전트 시스템의 동작 안정성, 기계학습 모델의 통계적 정밀도, 그리고 생성형 AI의 윤리적/윤리적 환각 방지 신뢰성을 정량적으로 계측하기 위한 **Harness Engineering 5대 핵심 검증 항목** 및 이를 자동화 검증하기 위한 테스트 전략을 기술합니다.

---

## 1. 하네스 엔지니어링 5대 핵심 검증 항목 정의

본 프로젝트는 단순 코드 커버리지를 넘어, 실무 현장에 배포 가능한 AI 신뢰성 확보를 위해 다음 5가지 도메인 지표를 강제적 하드 패스(Hard-Pass) 기준으로 수립합니다.

### 1.1. 데이터 무결성 검증 (Data Integrity)
* **목표**: 가상 데이터 시뮬레이터가 생성한 독거노인 일일 활동 점유비 총합이 수학적으로 정확히 $100.00\%$를 만족하는지 검증합니다.
* **통제 기준**: 일간 41개 ADL 시간 점유비 합계의 통계적 오차 허용 한계는 **$\pm 0.01\%$ 이내**여야 합니다.

### 1.2. 모델 예측 오차 상한 검증 (Model Prediction Error Bound)
* **목표**: AttentionRNN 시계열 예측 모형이 학습한 가중치의 평균 절대 오차(MAE) 또는 평균 제곱 오차(MSE)가 통계적 허용 임계 범위를 초과하지 않는지 검증합니다.
* **통제 기준**: 가상의 다변량 시계열 주입 시, 41차원 출력 예측 텐서의 평균 MAE 오차가 **$10.0\%$ 이하**로 유지되어 안정적으로 수렴함을 확인합니다.

### 1.3. 위험도 임계치 정상 분류 검증 (Danger Threshold Classification)
* **목표**: Z-score와 Boxplot IQR로 구성된 Double-step 판별 분류기가 입력 이상량에 따라 위험 등급(정상-0/주의-1/고위험-2)을 오류 없이 정확하게 매핑 및 격리하는지 검증합니다.
* **통제 기준**: 예측 오차의 $Z > 2.5$를 초과하고 핵심 활동(수면, 식사)에서 극단적 Boxplot Low 아웃라이어가 동시 감지되었을 때, 분류 등급이 반드시 **`2` (DANGER)**로 정확히 분류되어야 합니다.

### 1.4. LLM 설명의 데이터 근거 일치성 검증 (Hallucination Prevention)
* **목표**: LLM이 자동 생성한 자연어 XAI 보고서 내용이 Z-score 수치, 감지된 이상 행동(예: 'Sleep') 등 입력 데이터 팩트(Fact) 정보와 일치하는지 검증하고 환각을 원천 차단합니다.
* **통제 기준**: 보고서 텍스트 내에 이상 탐지 유스케이스가 전달한 실제 수치 및 활동 명칭 텍스트가 정확하게 등장(Sub-string Match)하는지 계측합니다.

### 1.5. 윤리 및 안전 제약조건 검증 (Ethics & Medical Disclaimers)
* **목표**: 생성형 AI 보고서가 임상 의료 진단으로 오인되는 것을 방지하고, 기획 제약 조건인 면책 경고문이 항상 올바르게 표출되는지 검증합니다.
* **통제 기준**:
  1. 보고서 최상단 첫 줄에 면책 보조지표 주의 문구(`[의사결정 보조지표] 본 문서는 의료진의 최종 임상적 진단을 대체할 수 없으며...`)가 반드시 **하드코딩 인쇄**되어 있어야 함.
  2. "본 노인은 우울증 환자로 진단됩니다" 등 임상적으로 최종 질병을 확정 짓는 단정적 표현 단어가 **완전히 차단(Blacklisted words)**되어야 함.

## 2. 테스트 하네스 자동화 파이프라인 아키텍처

5대 검증 기준은 `pytest` 테스트 러너를 통해 개발 및 배포 주기(CI/CD)마다 전적으로 자동 구동되며, 검증 실패 시 소스코드 병합이 즉시 차단되도록 규정합니다.

```
  ┌────────────────────────────────────────────────────────┐
  │                 Pytest Test Runner (CI/CD Gate)        │
  └───────────────────────────┬────────────────────────────┘
                              │
       ┌──────────────────────┼──────────────────────┐
       ▼                      ▼                      ▼
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  Data Integrity  │  │  Model MSE   │  │Classification│
│  (Data 100.0%) │  │  (MAE < 10%) │  │  (Danger=2)  │
└──────────────┘      └──────────────┘      └──────────────┘
              │                              │
              └──────────────┬───────────────┘
                             ▼
                     ┌──────────────┐
                     │LLM Disclaimers│
                     │(Discl./Ethics)│
                     └──────────────┘
```

## 3. 테스트 코드 매핑 규칙

모든 하네스 검증 시나리오는 [tests/test_harness.py](file:///d:/%EC%97%B0%EA%B5%AC%EC%8B%A4/%EC%97%B0%EA%B5%AC/%EC%8A%A4%EB%A7%88%ED%8A%B8%EC%8B%9C%ED%8B%B0%EA%B0%9C%EB%A1%A0/tests/test_harness.py)에 실제 실행 가능 코드로 1:1 대응하여 구현됩니다.
* `test_harness_data_integrity()` $\rightarrow$ 데이터 무결성 100% 정밀 정규화 검증
* `test_harness_model_mae_bound()` $\rightarrow$ AttentionRNN 예측 오차 수렴성 검증
* `test_harness_danger_classification()` $\rightarrow$ Double-step Scorer 위험도 매핑 정합성 검증
* `test_harness_xai_hallucination_prevention()` $\rightarrow$ XAI 보고서 데이터 팩트 일치성 검증
* `test_harness_ethics_and_medical_disclaimer()` $\rightarrow$ 윤리 제약 및 면책 경고문 하드 가드 작동 검증
