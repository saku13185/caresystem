# 기능 및 비기능 요구사항 명세서 (Requirements Specification)

본 문서는 예방적 돌봄 AI 에이전트 시스템의 명확한 개발 범위와 품질 표준을 정의하기 위해, 시스템 목표 및 이해관계자 분석을 기반으로 기능 요구사항(FR)과 비기능 요구사항(NFR), 그리고 BDD(Behavior-Driven Development) 기반의 수용 기준을 구체적이고 계측 가능한 수치로 분해하여 기록합니다.

---

## 1. 유스케이스 및 데이터 예외 흐름 (Use Case & Exception Flows)

### 1.1. 예방적 돌봄 분석 배치 실행 (기본 흐름)
1. **[시작]**: 일 단위 배치 스케줄러가 매일 자정 구동을 감지합니다.
2. **[합성 데이터 로드]**: 최근 16일 차까지의 일간 41개 ADL 시간 점유율(%) 데이터를 파일 저장소에서 수집합니다.
3. **[예측 및 비교]**: 15일간의 행동 데이터를 AttentionRNN 시계열 예측기에 주입하여 16일 차의 예측 점유율을 추론하고, 실제 16일 차의 점유율 데이터와 대조합니다.
4. **[이상 분류]**: Double-step 분석(Z-score 및 활동별 Boxplot 범위 검증)을 통해 대상 노인의 최종 위험도 등급(정상, 주의, 고위험)을 산출합니다.
5. **[보고서 생성]**: 판정 등급이 '주의' 또는 '고위험'인 경우, LLM XAI 추론기를 통해 자연어 보고서를 생성하여 대시보드 데이터베이스에 적재합니다.
6. **[종료]**: 사회복지사 대시보드 화면에 해당 알림 및 리포트 카드를 업데이트합니다.

### 1.2. 예외 및 대안 흐름 (Exception & Alternative Flows)
* **[EXCEPTION-01] 센서 데이터 단선 및 일간 결측치 발생**:
  * *현상*: 합성 데이터 생성 오류 또는 데이터 파이프라인 누수로 인해 특정 일자의 특정 활동 점유율 값이 `Null` 또는 $0\%$로 극단 유실된 경우.
  * *대응 로직 (Imputation)*: 시스템은 즉시 최근 3일간의 해당 노인의 동일 활동 점유율의 가중 평균값(최근일 가중치 $0.5$, 이전 2일 각각 $0.25$)으로 결측치를 자동 대체(Imputation)하고 경고 로그를 남긴 후 기본 흐름을 계속 진행합니다.
* **[EXCEPTION-02] LLM 추론 타임아웃 또는 API 오류**:
  * *현상*: 외부 LLM API 서비스 지연으로 인해 XAI 자연어 보고서 생성이 실패하거나 타임아웃이 발생한 경우.
  * *대응 로직 (Fallback)*: 자연어 설명 리포트는 빈칸으로 두되, 수치 분석 데이터(Z-score 및 예측 편차 활동 항목)에 기초한 '규칙 기반(Rule-based) 정형 설명 구문'(예: "[경고] 최근 수면 활동의 편차가 Z-score 2.7로 감지되어 고위험군으로 격리되었습니다. 상세 분석은 일시적으로 누락되었습니다.")을 긴급 적용하여 대시보드에 표출합니다.

## 2. 기능 요구사항 (Functional Requirements - FR)

### 2.1. FR-01: 시계열 합성 데이터 제너레이터 (Synthetic Data Generator)
* **FR-01-01**: 글로벌 벤치마크인 CASAS 데이터셋 구조를 모사하여, 41개 핵심 일상생활 행동(ADL: Bathe, Cook, Sleep, Wash, Eat, Watch_TV 등)의 일간 시간 점유율(%) 시계열 데이터를 파일 형태로 동적 생성해야 합니다.
* **FR-01-02**: 생성되는 임의의 1일에 대한 41개 ADL 시간 점유율의 합산은 반드시 정확히 $100.00\%$로 정규화(Normalization)되어야 합니다.
* **FR-01-03**: 파라미터 제어를 통해 정상적인 일상 노이즈 수준(점유율 변동 $\pm 5\%$) 외에, 특정 질병 시나리오(수면 장애로 인한 Sleep 시간 급감, 식사 거부로 인한 Cook/Eat 점유율 극소화 등)를 특정 시점에 강제로 주입할 수 있어야 합니다.

### 2.2. FR-02: AttentionRNN 기반 시계열 예측 엔진 (AttentionRNN Predictor)
* **FR-02-01**: 과거 15일간의 다변량 일간 ADL 점유율 윈도우 데이터(입력 텐서 규격: `(Batch, 15, 41)`)를 학습 및 추론의 입력으로 삼아야 합니다.
* **FR-02-02**: 입력받은 15일치 데이터를 기반으로 16일 차에 나타날 41개 활동 점유율의 예측 텐서(출력 규격: `(Batch, 41)`)를 정확히 예측 및 추론해야 합니다.
* **FR-02-03**: Self-Attention 메커니즘을 내장하여, 15일간의 윈도우 슬라이스 중 16일 차 행동 예측에 지대한 공헌을 한 주요 분석 기여 날짜의 가중치 배열(`(Batch, 15)`)을 매 회 추론 시점마다 모델 메모리에서 정량적으로 추출해야 합니다.

### 2.3. FR-03: Double-step 이상 탐지 및 Z-score 분류기 (Anomaly Classifier)
* **FR-03-01 (1단계 Z-score 판정)**: 16일 차 예측 텐서와 실측 텐서 간의 평균절대오차(MAE)를 계산하고, 역사적 데이터 평균 MAE($\mu_{MAE}$) 및 표준편차($\sigma_{MAE}$) 분포를 기준으로 개별 윈도우의 최종 오차 Z-score를 계량 연산해야 합니다.
  $$Z = \frac{MAE - \mu_{MAE}}{\sigma_{MAE}}$$
* **FR-03-02 (2단계 Boxplot 판정)**: Z-score가 경고 임계값($Z > 1.5$)을 초과하는 윈도우에 대해, 41개 개별 활동 점유율의 편차가 과거 동일 노인의 활동별 역사적 Boxplot 이상치 영역(Q1 - 1.5*IQR 미만 또는 Q3 + 1.5*IQR 초과)을 위반하는지 교차 분석해야 합니다.
* **FR-03-03**: 이중 검증 결과를 종합하여 최종 상태 코드를 **정상(Normal - 0)**, **주의(Warning - 1)**, **고위험(Danger - 2)**의 3단계 등급 구조 패킷으로 매핑하고 아웃풋 인터페이스로 출력해야 합니다.

### 2.4. FR-04: LLM XAI 자연어 리포트 생성기 (XAI Report Generator)
* **FR-04-01**: 최종 위험도 등급(주의/고위험), Z-score 값, Boxplot을 돌파한 특정 이상 활동 지표, 그리고 Attention 기여도가 가장 높은 날짜의 활동 정보를 취합하여 프롬프트 템플릿의 Context 블록으로 안전하게 주입(Context Injection)해야 합니다.
* **FR-04-02**: 사회복지사 및 생활지원사가 쉽게 해석하고 조치할 수 있도록 "수면 시간이 과거 평균 대비 3.2시간 감소했으며, 이는 8일 전 발생했던 불면 징후와 높은 연관성을 보입니다."와 같은 형태의 구체적이고 설득력 있는 한국어 설명 리포트를 자동 완성해야 합니다.

## 3. 비기능 요구사항 (Non-Functional Requirements - NFR)

### 3.1. NFR-01: 성능 및 추론 시간 (Performance)
* **NFR-01-01**: AttentionRNN 기반 예측 엔진의 1회 배치 윈도우 추론 시간은 인프라 CPU(Single Core) 기준 **100ms 이내**로 완료되어야 합니다.
* **NFR-01-02**: LLM XAI 자연어 보고서 생성 속도는 상용 클라우드 API 호출 시 1회 요청당 **5초 이내**, 로컬 서버 오픈소스 LLM(예: 7B급) 추론 시 **15초 이내**로 응답을 반환하고 출력을 완수해야 합니다.

### 3.2. NFR-02: 데이터 무결성 및 보정 (Data Integrity)
* **NFR-02-01**: 합성 데이터 제너레이터가 출력한 일간 점유율 총합에 대한 통계적 정밀도 오차는 소수점 4째짜리 수준인 **$\pm 0.01\%$ 이내**여야 하며, 이를 상회할 시 데이터 정규화 강제 보정 함수가 동작하여 총합 $100.00\%$를 무조건 보장해야 합니다.
* **NFR-02-02**: 데이터 누수나 누락으로 인한 Null 값 발생 시, 시스템 다운타임을 차단하기 위한 예외 처리(Imputation) 모듈이 반드시 작동하여 $100\%$ 데이터 정합성을 항시 유지한 채 추론 엔진으로 전달해야 합니다.

### 3.3. NFR-03: 윤리적 제약 및 보안 정책 (Ethics & Security)
* **NFR-03-01 (의료 오인 방지 경고)**: 자동 생성되는 모든 XAI 자연어 보고서 문서의 최상단 헤더(Header) 영역에 **"[의사결정 보조지표] 본 문서는 의료진의 최종 임상적 진단을 대체할 수 없으며, 예방 및 모니터링 보조 목적으로만 활용해야 합니다."**라는 주의 문구를 필수적으로 강제 하드코딩 표출해야 합니다.
* **NFR-03-02 (개인정보 차단)**: 딥러닝 예측 엔진 및 LLM 프롬프트에 제공되는 모든 데이터 스트림에 사회복지사/노인의 실제 성명, 연락처, 주민등록번호 등의 고유 개인정보(PII) 주입을 원천 차단하고 가상 식별자(UUID v4)로 완전 격리 대체하여 운영해야 합니다.

## 4. BDD 기반 수용 기준 (Acceptance Criteria)

### Scenario 1: 합성 데이터 일간 시간 점유율 합산의 정량적 정규화 검증
* **Given**: 합성 데이터 생성기가 특정 독거노인의 불면증 시나리오 파라미터를 입력받아 41개 핵심 ADL 일간 시간 점유율(%) 시계열 데이터를 30일치 동적 생성했을 때
* **When**: 생성된 파일 내 1일 차부터 30일 차까지 각 날짜별 41개 활동 점유율 백분율 수치의 합산값을 검증하는 정합성 스크립트를 구동했을 때
* **Then**: 모든 일자에 대한 활동 점유율 합이 오차 범위를 만족하는 **정확히 $100.00\%$ ($\pm 0.01\%$)**로 정상 보정되고 에러가 반환되지 않아야 합니다.

### Scenario 2: AttentionRNN의 15일 시계열 데이터 윈도우 추론 검증
* **Given**: 데이터 파이프라인 상에 특정 노인의 15일간의 정상적인 일간 시간 점유율(Shape: `(1, 15, 41)`) 데이터셋이 구비되어 있고, AttentionRNN 예측 모듈이 구동 준비 완료 상태일 때
* **When**: 해당 15일간의 다변량 윈도우 데이터를 모델에 주입하여 추론을 실행했을 때
* **Then**: 16일 차의 41개 활동 예측치 텐서(Shape: `(1, 41)`)와 각 15일에 대한 가중치를 매핑하는 Self-Attention 기여도 가중치 행렬(Shape: `(1, 15)`)이 에러 없이 동시 출력되어야 합니다.

### Scenario 3: Double-step 분류기의 고위험 진단 및 LLM XAI 보고서 자동 연계 검증
* **Given**: 16일 차 예측-실측 간 오차 Z-score가 $2.5$를 초과하고(고위험 기준), 2단계 Boxplot 검증 결과 "수면(Sleep)" 활동의 시간 점유율이 역사적 임계 범위인 IQR 아웃라이어를 크게 하회하는 수치 편차를 보였을 때
* **When**: Double-step 분류기가 이 진단 결과 패킷(위험 등급: 2, Z-score: 2.7, 이상 활동: Sleep)을 구성하여 LLM XAI 리포트 추론기에 컨텍스트로 전달하고 구성을 지시했을 때
* **Then**: 생성된 한글 보고서 내에 "위험도 등급: 고위험", "수면 감소"에 대한 구체적 텍스트 설명이 정상 포함되어야 하며, 문서 최상단에 "의료적 최종 진단 대체 불가" 경고문이 정확히 포함된 문서가 최종 출력되어야 합니다.
