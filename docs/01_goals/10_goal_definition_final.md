# 시스템 목표 정의서 (Goal Definition)

## 1. 문제 정의 (Problem Definition)

### 1.1. 사후 대응 위주 시스템의 한계
현재 독거노인을 위한 공공 및 민간 돌봄 서비스는 낙상, 급성 심정지 등 심각한 사고가 발생한 이후 비상벨이나 센서를 통해 신고가 접수되는 **사후 대응(Reactive)** 중심의 아키텍처를 가집니다. 이는 사고 발생 시점과 발견 시점 사이의 시간 격차로 인해 골든타임을 놓칠 위험이 큽니다.

### 1.2. 돌봄 공백의 존재
생활지원사나 사회복지사의 주기적인 방문 및 유선 안부 확인 서비스는 주 1~2회 수준에 머무르고 있습니다. 방문과 방문 사이, 혹은 전화 통화 사이의 시간 동안 독거노인의 일상 행동 패턴(식사, 수면, 위생 등)에 발생한 급격하거나 미세한 변화를 실시간으로 감지할 수 없는 **돌봄 공백**이 존재합니다.

### 1.3. 실제 센서 데이터 부재와 합성 데이터(Synthetic Data)의 필요성
지능형 예방 돌봄 시스템 개발을 위해서는 독거노인의 상세 일상 행동 로그 데이터가 필수적입니다. 그러나 실제 스마트홈 환경에 물리적인 센서를 대규모로 설치하는 것은 비용, 기술적 유지보수, 그리고 거주자의 프라이버시 침해 우려로 인해 현실적으로 불가능합니다. 
따라서, 글로벌 벤치마크 표준인 CASAS 데이터셋 규격(Timestamp, Sensor Location, Status, Activity)을 정밀하게 모사하여, 41개 핵심 일상 활동(예: Bathe, Cook, Sleep, Wash 등)의 일간 시간 점유율(%)을 동적으로 생성할 수 있는 **시계열 합성 데이터(Synthetic Data) 제너레이터** 구축이 선행되어야 합니다.

## 2. 목표 설정 (Goal Setting)

### 2.1. 서비스 목표 (사용자 가치 중심)
* **예방적 돌봄(Proactive Care) 실현**: 독거노인의 일상생활(ADL) 패턴 변화를 장기적으로 추적하여 고독사나 급격한 건강 악화의 전조 증상을 골든타임 내에 조기 감지합니다.
* **설명 가능한 돌봄(Explainable Care) 제공**: 단순한 수치나 위험 등급 제시를 넘어, "왜 이 노인이 현재 위험 상태로 의심되는가"를 사회복지사가 즉각 이해할 수 있도록 쉬운 자연어로 설명하여 돌봄의 질을 높입니다.
* **업무 효율성 극대화**: 제한된 인력의 사회복지사들이 수많은 대상자 중 우선적으로 확인해야 할 대상자를 직관적으로 선별할 수 있도록 지원합니다.

### 2.2. 시스템 목표 (기술 파이프라인 중심)
* **시계열 딥러닝 예측 엔진**:
  * 과거 15일간의 일상생활(ADL) 활동 점유율 데이터를 입력으로 받아, 다음 날(16일 차)의 41개 활동 점유율 분포를 정밀하게 예측하는 **Attention-based RNN(AttentionRNN)** 모델을 구축합니다.
* **Z-score 기반 Double-step 분류 파이프라인**:
  * 예측된 행동 점유율 분포와 실제 발생한 16일 차 행동 데이터 간의 평균절대오차(MAE)를 계산합니다.
  * 계산된 오차를 Z-score로 변환하여 1차 이상치를 판별하고, 각 활동별 역사적 분포(Boxplot) 임계치를 벗어나는지 확인하는 Double-step 이상 탐지 로직을 설계하여 **정상 / 주의 / 고위험**의 3단계 위험 등급을 산정합니다.
* **LLM 기반 설명 가능성(XAI) 추론 파이프라인**:
  * 이상 탐지 스코어, 위험도 등급, 그리고 편차가 크게 발생한 특정 활동 데이터를 컨텍스트로 LLM에 주입하여, 사회복지사가 즉각 이해하고 조치를 취할 수 있는 한글 리포트를 자동 생성합니다.
  * 프롬프트 엔지니어링 및 철저한 가이드라인 제어를 통해 LLM의 정보 환각(Hallucination) 현상을 차단합니다.

## 3. 제약 조건 (Engineering Constraints)

* **비의료 보조 시스템 선언**: 본 시스템이 도출하는 이상 탐지 점수 및 LLM 리포트는 어떠한 경우에도 임상적 진단이나 의료적 최종 판정으로 표현되어서는 안 되며, 오직 사회복지사의 수동 모니터링을 보조하는 **"의사결정 지원 지표"**로 엄격히 한정합니다.
* **데이터 격리 및 보안**: 생성된 가상 ADL 데이터는 비식별화된 상태를 유지해야 하며, 시스템 외부로 불필요한 민감 정보가 유출되지 않도록 설계합니다.
* **아키텍처 제약**: 본 문서 단계에서는 물리적 데이터베이스 스키마 설계, 구체적인 API 명세, 사용자 화면 UI 레이아웃, 혹은 세부적인 기능 요구사항(FR) 목록을 다루지 않고 순수 시스템 목표 수립에만 집중합니다.
