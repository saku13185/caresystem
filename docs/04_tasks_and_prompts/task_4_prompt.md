## 2. 디자인 및 기능 컴포넌트 필수 요건

* **Wow Factor 시각 디자인 및 HSL 컬러 칩**:
  - HSL 미학적 팔레트를 반영하여 모던하고 신뢰성 높은 **다크 모드 글라스모피즘(Glassmorphism)** 레이아웃 스타일링을 적용해야 합니다. (Plain RGB red/blue 배제, harmonized HSL-based alert colors 적용).
* **3대 핵심 UI 위젯 기능**:
  1. **위험 등급별 대상 노인 보드**: 정상, 주의, 고위험 카드 리스트 필터 제공.
  2. **어텐션 및 오차 트렌드 차트**: Attention 가중치(15일 기여 날짜)와 예측 편차가 큰 상위 ADL 항목을 Streamlit 기본 Line/Bar chart 또는 Plotly를 사용하여 미려하게 시각화 렌더링.
  3. **XAI 한글 리포트 팝업 & 피드백 창구**: LLM이 자동 생성한 돌봄 리포트를 팝업 카드로 표출하고, 사회복지사가 분석 결과에 대해 '정탐(승인 및 보호자 경보 발송)' / '오탐(반려)'을 결정하여 상세 피드백 메시지와 조치록 메모를 기입할 수 있는 피드백 폼 제공.

## 3. 에러 처리 및 데이터 영속화 규칙

* **피드백 데이터 영속성 바인딩**: 복지사가 피드백 입력 후 '저장/승인' 버튼을 클릭하면, 대시보드는 즉시 SQLite 데이터베이스(또는 저장성 CSV 영속 파일) 내 `caregiver_alerts` 테이블의 `action_status`와 `feedback_message` 컬럼에 실시간 SQL UPDATE 쿼리를 날려 예외 없이 완벽하게 영속성 저장을 마쳐야 합니다.
* **DB 잠김 오류 방지**: 쓰기 연산 시 SQLite 다중 세션 충돌 방지를 위해 단일 전역 DB Connection 인스턴스 또는 Lock 세션을 사용하여 대시보드가 불필요하게 셧다운되는 예외를 완벽히 통제 차단합니다.

---

## 4. UI 통합 검증 수용 기준 (Manual/Auto Verification)

* **수용 기준 (UI Integration Criteria)**:
  - **Given**: 사회복지사 웹 브라우저 상에 Streamlit 대시보드 화면이 구동되어 있고 '고위험군' 노인 카드가 노출되어 있을 때
  - **When**: 사회복지사가 특정 노인의 카드를 클릭하고 팝업창에서 '오탐(Reject)' 라디오 버튼을 선택한 후 피드백 메모란에 "단순 주말 외출 시간 급증으로 오탐 감지되어 반려함"을 기입하고 '피드백 저장 및 경보 반려' 버튼을 클릭했을 때
  - **Then**: 데이터베이스 `caregiver_alerts` 테이블 상의 해당 경보 상태값(`action_status`)이 정확히 `'REJECTED'`로 업데이트되고 메모가 영속 기록되며, 화면 상에 "반려 피드백이 전송되었습니다"라는 녹색 알림 배너가 렌더링되어야 합니다.
