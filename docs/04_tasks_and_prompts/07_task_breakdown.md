# 세부 스프린트 백로그 작업 분해 정의서 (Task Breakdown)

본 문서는 예방적 돌봄 AI 에이전트 시스템의 전체 아키텍처 및 물리 데이터베이스 설계를 바탕으로, 독립적으로 개발, 검증 및 통합 가능한 최소 단위의 스프린트 백로그 작업(Task)을 정의하고 각 작업 간의 선후 의존 관계(Blocker)를 분석하여 체계적인 개발 로드맵을 구축하기 위해 작성되었습니다.

## 2. 작업별 세부 마일스톤 및 설계 명세

### 2.1. Task 1: DB 물리 스키마 세팅 및 가상 데이터 시뮬레이터 개발
* **담당 부서/역할**: 데이터 엔지니어 & 가상화 시뮬레이션 개발자
* **세부 작업**:
  1. `src/domain/resident.py` 내의 엔터티 사양을 충족하는 SQLite 물리 테이블 DDL 스크립트 작성 및 커넥터 클래스 구현 (`src/infrastructure/persistence/`).
  2. CASAS 표준 데이터셋을 모사하여 41개 ADL 일간 시간 점유율(%) 합성 시계열 데이터를 무작위/난수 난이도로 동적 생성하는 파이프라인 구축 (`src/infrastructure/generators/`).
  3. 불면증, 식사 거부 등의 특정 질병 패턴 시나리오를 주입하기 위해, 특정 날짜 구간에 특정 활동(수면, 식사 등)의 백분율 점유율을 급감/급증시키는 파라미터 제어 함수 구현.
  4. 일간 활동 총합이 소수점 오차를 돌파할 시 강제로 합산 $100.00\%$로 수렴 스무딩 시키는 정규화 알고리즘 탑재.
* **검증 계획**: `tests/test_generators.py`를 통해 100일치 생성 데이터의 일별 백분율 총합이 정확히 오차 범위 내인 $100.00\%$를 만족하는지 자동 검증 테스트 구동.

### 2.2. Task 2: ADL 전처리 및 AttentionRNN 시계열 예측 엔진 구현
* **담당 부서/역할**: 시계열 딥러닝 AI 엔지니어
* **세부 작업**:
  1. 데이터 로딩 시 결측치(Null, 유실)를 발견했을 때 작동하는 예외 보강 모듈 구축. 최근 3일간의 해당 노인 ADL 점유율 가중평균값($0.5$, $0.25$, $0.25$)으로 결측 데이터를 보충하는 파이프라인 설계 (`src/usecases/preprocess_adl_data.py`).
  2. `PyTorch`를 사용하여 입력 크기 $41$, 숨겨진 차원 $64$, 윈도우 크기 $15$를 가지는 Attention-based GRU/LSTM(AttentionRNN) 신경망 아키텍처 정의 (`src/infrastructure/models/`).
  3. 훈련용 합성 데이터셋으로 모델을 피팅(Fitting)하고 가중치 파일(`.pt`)을 영속 저장하는 학습 파이프라인 구현.
  4. 추론 시점의 어텐션 모듈 가중치 맵 `(Batch, 15)`을 정량적으로 추출하는 API 포트 설계.
* **검증 계획**: `tests/test_models.py`에서 임의의 `(1, 15, 41)` 입력 주입 시 출력 예측 텐서 크기가 `(1, 41)`이고 Attention 텐서 크기가 `(1, 15)` 인지 형태(Shape) 정합성 확인.

### 2.3. Task 3: Double-step 분류기 및 LLM XAI 리포팅 엔진 연동
* **담당 부서/역할**: 백엔드 알고리즘 & LLM 에이전트 개발자
* **세부 작업**:
  1. 예측치와 실측치 MAE 오차의 Z-score 통계 연산 모듈 구축 및 Boxplot IQR 아웃라이어 아웃 범위 대조 로직 구현 (`src/infrastructure/scorers/`).
  2. Z-score 임계치($Z>2.5$) 및 Boxplot 이탈 정보를 바탕으로 최종 위험도 정상(0), 주의(1), 고위험(2)의 상태 코드로 매핑하는 이중 분류 오케스트레이터 설계.
  3. Z-score 값, Boxplot 이탈 활동, 어텐션 가중치 최고 기여도 날짜 정보를 조합하여 프롬프트의 Context 블록으로 자동 구성하는 템플릿 인젝터 구현 (`src/infrastructure/llm/`).
  4. LLM API 클라이언트를 연결하여 한국어 자연어 보고서 텍스트 도출. 보고서 최상단에 **"[의사결정 보조지표] 본 문서는 의료진의 최종 진단을 대체할 수 없습니다."** 주의보 경고문을 강제 출력하고 노인의 개인정보는 UUID v4 격리 매핑 유지.
* **검증 계획**: 모의 주의/고위험 지표 인입 시 LLM XAI 텍스트 파일 내에 경고 문구 및 등급 명칭이 정상 포함되는지 단언문(Assert) 검증 수행.

### 2.4. Task 4: Streamlit 기반 복지사 안심 웹 대시보드 UI 구축
* **담당 부서/역할**: 프론트엔드 UI 개발자
* **세부 작업**:
  1. `Streamlit` 프레임워크를 구동하고 메인 반응형 웹 대시보드 인터페이스 코딩 (`src/presentation/app.py`).
  2. 대시보드 메인에 위험도 순으로 대상 노인 리스트(정상/주의/고위험)를 직관적으로 카드 렌더링.
  3. 특정 노인 클릭 시, 15일간의 Attention 가중치 기여 날짜 변동 및 실제-예측 편차 차트(Line/Bar chart) 시각화.
  4. LLM이 생성한 한글 XAI 예방 리포트 팝업 카드 표출 및 해당 분석 결과가 맞는지 사회복지사가 피드백할 수 있는 '정탐/오탐 여부' 라디오 버튼과 '메모 메모 조치사항' 피드백 기입 창구 개설 및 SQLite DB 업데이트 연동.
* **검증 계획**: Streamlit 로컬 구동 테스트 및 복지사 피드백 데이터 기입 후 SQLite DB 내 `caregiver_alerts` 테이블의 `action_status` 및 `feedback_message`가 정확히 업데이트되었는지 현장 정합성 수동 검사 수행.
